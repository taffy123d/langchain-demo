# Python代码格式化规范

## 核心原则
1. 遵循PEP 8基本规范，但不依赖外部格式化工具
2. 保持代码一致性，修复明显的格式问题
3. 不改变代码逻辑，仅调整格式

## 格式规范细则

### 1. 缩进与空格
- 使用4个空格进行缩进（禁止使用Tab）
- 函数定义、类定义、控制结构后加空格
- 运算符两侧加空格：`a = b + c`
- 逗号后加空格：`func(a, b, c)`
- 冒号后加空格（字典、切片除外）

### 2. 行长度与换行
- 最大行长度建议79字符（可适当放宽）
- 过长的表达式在适当位置换行
- 使用括号进行隐式行连接

### 3. 导入规范
- 导入语句分组顺序：标准库、第三方库、本地模块
- 每组之间用空行分隔
- 避免通配符导入（`from module import *`）

### 4. 命名规范
- 函数和变量：小写字母，单词间用下划线连接（snake_case）
- 类名：每个单词首字母大写（CamelCase）
- 常量：全大写字母，单词间用下划线连接

### 5. 注释格式
- 行注释使用`#`，注释前加一个空格
- 块注释使用多个`#`，与代码缩进对齐
- 文档字符串使用三重引号

## 执行步骤
1. 读取用户指定的代码文件或当前目录的Python文件
2. 分析代码格式问题，识别不符合规范的地方
3. 按照上述规范逐步修复格式问题
4. 输出格式化后的代码，并说明所做的修改

## 注意事项
- 仅修改格式，不改变代码逻辑和功能
- 优先处理用户明确指定的文件
- 对于复杂格式问题，提供修改建议而非强制修改
- 保持备份意识，避免意外数据丢失
